# Getting Started with Specra

This guide will get you from zero to a fully functional documentation site in under 5 minutes. We'll cover installation, basic configuration, and writing your first page.

## Prerequisites

Before you begin, make sure you have:
- **Node.js 18 or higher** (check with `node --version`)
- A package manager: **npm** (comes with Node.js), **yarn**, or **pnpm**
- A code editor (VS Code recommended for MDX support)

## Installation

The fastest way to get started is using `create-specra`, our scaffolding tool that sets up everything for you.

<Steps>
  <Step title="Create Your Project">
    Run the create command with your project name:

    <Tabs defaultValue="npm">
      <Tab label="npm">
```bash
npx create-specra@latest my-docs
```
      </Tab>
      <Tab label="yarn">
```bash
yarn create specra my-docs
```
      </Tab>
      <Tab label="pnpm">
```bash
pnpm create specra my-docs
```
      </Tab>
    </Tabs>

    This creates a new directory called `my-docs` with all the files you need to get started.
  </Step>

  <Step title="Navigate and Install">
    Move into your new project and install dependencies:

    <Tabs defaultValue="npm">
      <Tab label="npm">
```bash
cd my-docs
npm install
```
      </Tab>
      <Tab label="yarn">
```bash
cd my-docs
yarn install
```
      </Tab>
      <Tab label="pnpm">
```bash
cd my-docs
pnpm install
```
      </Tab>
    </Tabs>

    Installation typically takes 30-60 seconds depending on your internet connection.
  </Step>

  <Step title="Start the Dev Server">
    Fire up the development server:

    <Tabs defaultValue="npm">
      <Tab label="npm">
```bash
npm run dev
```
      </Tab>
      <Tab label="yarn">
```bash
yarn dev
```
      </Tab>
      <Tab label="pnpm">
```bash
pnpm dev
```
      </Tab>
    </Tabs>

    Open [http://localhost:3000](http://localhost:3000) in your browser. You should see your documentation site with sample content!

    <Callout type="success">
    The dev server includes hot reload. Edit any MDX file and watch your changes appear instantly.
    </Callout>
  </Step>
</Steps>

## Project Structure

After running `create-specra`, here's what you get:

```txt
my-docs/
├── app/                     # Next.js App Router
│   ├── layout.tsx           # Root layout with Specra providers
│   ├── page.tsx             # Home page
│   └── docs/                # Documentation routes
│       └── [...slug]/       # Dynamic catch-all route
│           └── page.tsx     # Doc page renderer
├── docs/                    # Your documentation content
│   └── v1.0.0/             # Version folder
│       ├── about.mdx
│       ├── getting-started.mdx
│       └── guides/         # Folder grouping
│           ├── _category_.json
│           └── writing.mdx
├── public/                  # Static assets (images, icons)
├── specra.config.json      # Specra configuration
├── package.json
├── next.config.mjs         # Next.js config (auto-managed)
└── tailwind.config.ts      # Tailwind configuration
```

<Callout type="info">
**Important:** All documentation files must use the `.mdx` extension, not `.md`. MDX lets you use React components alongside Markdown.
</Callout>

## Your First Documentation Page

Let's create your first custom documentation page.

<Steps>
  <Step title="Create a New File">
    Inside `docs/v1.0.0/`, create a file called `hello.mdx`:

```mdx
---
title: Hello Specra
description: My first documentation page
sidebar_position: 3
---

# Hello Specra!

This is my first documentation page. I can use **bold text**,
*italic text*, and even [links](https://specra.dev).

## Features I Love

- Fast hot reload
- Beautiful components
- Simple file structure
- Zero configuration needed
```
  </Step>

  <Step title="View Your Page">
    Save the file and visit [http://localhost:3000/docs/v1.0.0/hello](http://localhost:3000/docs/v1.0.0/hello)

    You should see your new page in the sidebar and rendered beautifully in the main content area.
  </Step>

  <Step title="Add a Component">
    Let's make it more interesting with a Callout component:

```mdx
---
title: Hello Specra
description: My first documentation page
sidebar_position: 3
---

# Hello Specra!

This is my first documentation page.

<Callout type="success">
  I just added an interactive component to my Markdown! No imports needed.
</Callout>

## Next Steps

Ready to learn more? Check out the [Components](/docs/v1.0.0/components) section.
```

    Save and watch it update instantly. Notice how you didn't need to import the Callout component - it's automatically available in all your MDX files.
  </Step>
</Steps>

## Understanding Frontmatter

Frontmatter is the metadata at the top of each MDX file, wrapped in triple dashes (`---`). It controls how your page appears and behaves.

### Available Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `title` | string | ✅ Yes | Page title shown in sidebar, browser tab, and breadcrumbs |
| `description` | string | Recommended | Brief description for SEO and social media previews |
| `sidebar_position` | number | No | Controls sidebar order (lower numbers appear first) |
| `tab_group` | string | No | Assigns page to a tab group (like "guides", "api", "components") |
| `icon` | string | No | Lucide icon name to display next to title in sidebar |
| `tags` | array | No | Tags for categorization and filtering |
| `draft` | boolean | No | Set to `true` to hide page in production builds |
| `last_updated` | string | No | Override auto-detected last updated date |

### Example Frontmatter

```mdx
---
title: Advanced Configuration
description: Deep dive into Specra configuration options
sidebar_position: 10
tab_group: guides
icon: settings
tags: [configuration, advanced, customization]
draft: false
---

# Your content starts here
```

<Callout type="tip">
Pages without `sidebar_position` are sorted alphabetically. Use this field to create a specific reading order.
</Callout>

## Organizing with Folders

Group related pages into folders for better organization. Specra automatically generates navigation from your folder structure.

### Basic Folder Structure

```txt
docs/v1.0.0/
├── getting-started.mdx
├── guides/
│   ├── installation.mdx
│   ├── configuration.mdx
│   └── deployment.mdx
└── api/
    ├── overview.mdx
    └── endpoints.mdx
```

Each folder becomes a collapsible section in the sidebar.

### Category Configuration

Create a `_category_.json` file inside any folder to customize how it appears:

```json
{
  "label": "User Guides",
  "position": 2,
  "collapsible": true,
  "collapsed": false,
  "tab_group": "guides",
  "icon": "book-open"
}
```

**Options:**
- `label`: Display name in sidebar (defaults to folder name)
- `position`: Order among other folders/pages
- `collapsible`: Whether users can collapse the section
- `collapsed`: Initial collapsed state
- `tab_group`: Assign entire folder to a tab group
- `icon`: Lucide icon name

### Index Pages

Create an `index.mdx` file in a folder to provide a landing page when users click the folder name:

```txt
guides/
├── _category_.json
├── index.mdx          ← Landing page for /docs/v1.0.0/guides
├── installation.mdx
└── deployment.mdx
```

## Using Components

Specra comes with a rich library of components that are automatically available in all MDX files. No imports needed.

### Quick Examples

**Callouts for important information:**
```mdx
<Callout type="info">
  This is helpful information for your readers.
</Callout>

<Callout type="warning">
  Be careful with this setting!
</Callout>
```

**Tabs for multiple options:**
```mdx
<Tabs defaultValue="npm">
  <Tab label="npm">
    npm install specra
  </Tab>
  <Tab label="yarn">
    yarn add specra
  </Tab>
</Tabs>
```

**Steps for tutorials:**
```mdx
<Steps>
  <Step title="First Step">
    Do this first.
  </Step>
  <Step title="Second Step">
    Then do this.
  </Step>
</Steps>
```

**Cards for visual navigation:**
```mdx
<CardGrid cols={2}>
  <Card
    icon="zap"
    title="Quick Start"
    description="Get up and running fast"
    href="/docs/v1.0.0/getting-started"
  />
  <Card
    icon="book"
    title="Full Guide"
    description="Complete documentation"
    href="/docs/v1.0.0/guides"
  />
</CardGrid>
```

Explore all available components in the [Components](/docs/v1.0.0/components) section.

## Markdown Support

Specra supports all standard Markdown syntax plus GitHub Flavored Markdown extensions.

**Headings:** `#` for H1, `##` for H2, etc.
**Bold:** `**text**` or `__text__`
**Italic:** `*text*` or `_text_`
**Links:** `[text](url)`
**Images:** `![alt](url)`
**Code:** `` `inline code` ``
**Code blocks:** Triple backticks with language

**Tables:**
```markdown
| Column 1 | Column 2 |
|----------|----------|
| Data 1   | Data 2   |
```

**Task lists:**
```markdown
- [x] Completed task
- [ ] Pending task
```

**Blockquotes:**
```markdown
> This is a quote
```
