# Callout

Les Callouts attirent l'attention sur des informations importantes dans votre documentation. Ils existent en différents styles pour transmettre différents types de messages, des astuces utiles aux avertissements critiques.

## Utilisation de base

<Tabs defaultValue="Preview">
  <Tab label="Aperçu">
<Callout type="info">
  Ceci est une information utile pour le lecteur.
</Callout>
  </Tab>
  <Tab label="Code">
```mdx
<Callout type="info">
  Ceci est une information utile pour le lecteur.
</Callout>
```
  </Tab>
</Tabs>

## Props

| Prop | Type | Défaut | Description |
|------|------|--------|-------------|
| `type` | `string` | `"info"` | Style de callout : `info`, `note`, `tip`, `warning`, `success`, `error`, ou `danger` |
| `children` | `ReactNode` | Requis | Le contenu à l'intérieur du callout |

## Alertes de style GitHub

Vous préférez la syntaxe Markdown ? Specra prend également en charge la syntaxe d'alerte de style GitHub en utilisant des blockquotes :

<Tabs defaultValue="Preview">
  <Tab label="Aperçu">
> [!NOTE]
> Ceci est une note utilisant la syntaxe de style GitHub.

> [!TIP]
> Ceci est une astuce utilisant la syntaxe blockquote.

> [!WARNING]
> Ceci est une alerte d'avertissement.
  </Tab>
  <Tab label="Code">
```markdown
> [!NOTE]
> Ceci est une note utilisant la syntaxe de style GitHub.

> [!TIP]
> Ceci est une astuce utilisant la syntaxe blockquote.

> [!WARNING]
> Ceci est une alerte d'avertissement.
```
  </Tab>
</Tabs>

**Types d'alertes pris en charge :**
- `[!NOTE]` - Identique à type="note"
- `[!TIP]` - Identique à type="tip"
- `[!IMPORTANT]` - Identique à type="warning"
- `[!WARNING]` - Identique à type="warning"
- `[!CAUTION]` - Identique à type="danger"

## Quand utiliser chaque type

**Info** - Contexte supplémentaire qui améliore la compréhension
- "Vous pouvez également réaliser cela en utilisant le CLI"
- "Cette fonctionnalité est disponible dans la v2.0 et ultérieure"

**Note** - Informations importantes qui ne doivent pas être ignorées
- "Ce paramètre affecte tous les utilisateurs de votre organisation"
- "Les changements prennent effet après le redémarrage du serveur"

**Tip** - Meilleures pratiques et suggestions utiles
- "Pour de meilleures performances, activez la mise en cache"
- "Utilisez TypeScript pour une meilleure autocomplétion"

**Warning** - Problèmes potentiels ou changements de rupture
- "Cela modifiera votre schéma de base de données"
- "Obsolète dans la v3.0, utilisez newMethod() à la place"

**Success** - Confirmations positives
- "Configuration terminée ! Votre site est prêt"
- "Tous les tests ont réussi"

**Error** - Problèmes critiques à éviter
- "N'exposez pas votre clé API dans le code client"
- "Cette opération ne peut pas être annulée"

**Danger** - Actions destructrices ou irréversibles
- "Cela supprimera définitivement toutes les données"
- "Continuer révoquera tous les jetons d'accès"

## Accessibilité

Les callouts sont sémantiques et accessibles :

- Utilise les rôles ARIA appropriés
- Compatible avec les lecteurs d'écran
- Navigable au clavier
- Conforme au contraste des couleurs (WCAG AA)
- Les icônes sont décoratives avec un texte alternatif approprié
