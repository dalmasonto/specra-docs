# create-specra

`create-specra` is the official CLI tool for scaffolding new Specra documentation sites. It sets up everything you need in seconds: project structure, dependencies, configuration, and sample content.

## Quick Start

The fastest way to create a new documentation site:

<Tabs defaultValue="npm">
  <Tab label="npm">
```bash
npx create-specra@latest my-docs
```
  </Tab>
  <Tab label="yarn">
```bash
yarn create specra my-docs
```
  </Tab>
  <Tab label="pnpm">
```bash
pnpm create specra my-docs
```
  </Tab>
</Tabs>

This creates a new directory called `my-docs` with everything configured and ready to go.

## What Gets Created

When you run `create-specra`, here's what you get:

```txt
my-docs/
├── app/
│   ├── layout.tsx              # Root layout with Specra setup
│   ├── page.tsx                # Landing page
│   ├── globals.css             # Global styles
│   └── docs/
│       └── [...slug]/
│           └── page.tsx        # Documentation page renderer
├── docs/
│   └── v1.0.0/
│       ├── about.mdx           # Sample about page
│       ├── getting-started.mdx # Sample getting started guide
│       └── components/         # Component examples
├── public/
│   ├── icon-light-32x32.png   # Favicon (light mode)
│   └── icon-dark-32x32.png    # Favicon (dark mode)
├── specra.config.json          # Specra configuration
├── package.json                # Dependencies and scripts
├── next.config.mjs             # Next.js configuration
├── tailwind.config.ts          # Tailwind CSS config
├── tsconfig.json               # TypeScript config
├── .gitignore
└── README.md
```

## Interactive Setup

When you run `create-specra`, it may prompt you for:

**Project name** (if not provided as argument)
```bash
npx create-specra@latest
# Prompts: What is your project named?
```

**Package manager preference** (if multiple are detected)
```bash
# Will ask: Which package manager do you want to use?
# Options: npm, yarn, pnpm
```

The CLI detects what's available on your system and suggests the best option.

## Command Options

### Basic Usage

```bash
npx create-specra@latest [project-name] [options]
```

### Examples

**Create with a specific name:**
```bash
npx create-specra@latest my-awesome-docs
```

**Create in current directory:**
```bash
npx create-specra@latest .
```

**Use a specific package manager:**
```bash
npx create-specra@latest my-docs --use-npm
npx create-specra@latest my-docs --use-yarn
npx create-specra@latest my-docs --use-pnpm
```

## Default Configuration

The generated `specra.config.json` includes sensible defaults:

```json
{
  "site": {
    "title": "My Documentation",
    "description": "Documentation site built with Specra",
    "url": "https://example.com",
    "baseUrl": "/",
    "activeVersion": "v1.0.0"
  },
  "theme": {
    "defaultMode": "system",
    "respectPrefersColorScheme": true
  },
  "navigation": {
    "showSidebar": true,
    "collapsibleSidebar": true,
    "showBreadcrumbs": true,
    "showTableOfContents": true,
    "tocPosition": "right"
  },
  "features": {
    "showLastUpdated": true,
    "showReadingTime": true,
    "versioning": true

  "deployment": {
    "target": "vercel"
```

You can customize any of these settings. See [Configuration](/docs/v1.0.0/configuration/overview) for all available options.

## Available Scripts

The generated `package.json` includes these scripts:

| Script | Command | Description |
|--------|---------|-------------|
| `dev` | `next dev` | Start development server with hot reload |
| `build` | `next build` | Build for production (Vercel, Node.js hosting) |
| `start` | `next start` | Start production server (after `build`) |
| `build:export` | `next build && next export` | Build static site (GitHub Pages, Netlify) |
| `lint` | `next lint` | Run ESLint to check code quality |

## Troubleshooting

### Command not found

If you get "command not found" errors:

**Check Node.js version:**
```bash
node --version
# Should be 18.0.0 or higher
```

**Try with full npx path:**
```bash
npx create-specra@latest my-docs
```

### Permission errors

On Linux/macOS, you might need to adjust npm permissions:

```bash
sudo chown -R $USER ~/.npm
```

Or use a node version manager like [nvm](https://github.com/nvm-sh/nvm).

### Installation fails

If dependency installation fails:

1. Delete `node_modules` and lock file
2. Clear package manager cache
3. Try again

```bash
rm -rf node_modules package-lock.json
npm cache clean --force
npm install
```
