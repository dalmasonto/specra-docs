## Sobre este ejemplo

Este ejemplo muestra cómo el analizador maneja:

- ✅ Detección del **formato OpenAPI 3.0**
- ✅ Resolución de **referencias de esquema** (`$ref`)
- ✅ **Parámetros de ruta** (`{id}` convertido a `:id`)
- ✅ **Esquemas de solicitud/respuesta** con ejemplos
- ✅ **Esquemas de seguridad** (Tokens Bearer, claves API)
- ✅ **Múltiples códigos de respuesta** (éxito y errores)

## Usar su propia especificación OpenAPI

Para usar su propia especificación OpenAPI:

1. **Coloque su archivo de especificación** en el directorio `/public/api-specs/`
2. **Referéncielo en su archivo MDX**:

```mdx
<ApiReference spec="/api-specs/your-api.json" parser="openapi" />
```

O deje que detecte automáticamente:

```mdx
<ApiReference spec="/api-specs/your-api.json" />
```

El analizador soporta formatos **OpenAPI 3.x** y **Swagger 2.0**.
