# Advanced Configuration

Beyond the basics, Specra offers powerful features like analytics, search, custom footers, banners, and environment variables. This guide covers all the advanced configuration options.

## Analytics

Track how users interact with your documentation.

### Google Analytics

```json
{
  "analytics": {
    "googleAnalytics": "G-XXXXXXXXXX"
  }
}
```

Get your measurement ID from [Google Analytics](https://analytics.google.com).

### Google Tag Manager

```json
{
  "analytics": {
    "googleTagManager": "GTM-XXXXXXX"
  }
}
```

More flexible than Google Analytics, allows custom tracking without code changes.

### Plausible Analytics

Privacy-friendly alternative to Google Analytics:

```json
{
  "analytics": {
    "plausible": "docs.example.com"
  }
}
```

Sign up at [plausible.io](https://plausible.io).

### Multiple Analytics

Use several providers simultaneously:

```json
{
  "analytics": {
    "googleAnalytics": "G-XXXXXXXXXX",
    "plausible": "docs.example.com"
  }
}
```

## Search Configuration

Enable powerful search for your documentation.

### Basic Search Setup

```json
{
  "search": {
    "enabled": true,
    "placeholder": "Search documentation...",
    "provider": "meilisearch"
  }
}
```

### MeiliSearch

Full-featured search with typo tolerance:

```json
{
  "search": {
    "enabled": true,
    "provider": "meilisearch",
    "meilisearch": {
      "host": "https://search.example.com",
      "apiKey": "your-search-key",
      "indexName": "docs"
    }
  }
}
```

See [Search Setup](/docs/v1.0.0/search-setup) for complete MeiliSearch configuration.

### Local Search

Simple search without external dependencies:

```json
{
  "search": {
    "enabled": true,
    "provider": "local",
    "placeholder": "Search..."
  }
}
```

**Pros:**
- No setup required
- Works offline
- Free

**Cons:**
- Less powerful than MeiliSearch
- No typo tolerance
- Slower for large sites

## Footer Configuration

Customize your documentation footer.

### Basic Footer

```json
{
  "footer": {
    "copyright": "¬© 2024 My Company. All rights reserved."
  }
}
```

### Complete Footer

```json
{
  "footer": {
    "copyright": "¬© 2024 Acme Corp. All rights reserved.",
    "customContent": "<p>Need help? <a href='/support'>Contact support</a></p>",
    "links": [
      {
        "title": "Product",
        "items": [
          { "label": "Features", "href": "/features" },
          { "label": "Pricing", "href": "/pricing" },
          { "label": "Changelog", "href": "/changelog" }
        ]
      },
      {
        "title": "Resources",
        "items": [
          { "label": "Documentation", "href": "/docs" },
          { "label": "API Reference", "href": "/api" },
          { "label": "Examples", "href": "/examples" }
        ]
      },
      {
        "title": "Company",
        "items": [
          { "label": "About", "href": "/about" },
          { "label": "Blog", "href": "/blog" },
          { "label": "Careers", "href": "/careers" }
        ]
      }
    ],
    "branding": {
      "showBranding": true,
      "logo": "/brand-logo.svg",
      "title": "Powered by Specra",
      "url": "https://specra.dev"
    }
  }
}
```

### Footer Options

**copyright**
- Copyright notice displayed at bottom
- Supports HTML

**customContent**
- Custom HTML inserted in footer
- Use for extra messaging or links

**links**
- Organized link columns
- Each column has a title and items
- Items have label and href

**branding**
- Show "Powered by" branding
- Customize logo, title, and URL
- Set `showBranding: false` to hide

## Features Configuration

Enable or disable various documentation features.

```json
{
  "features": {
    "editUrl": "https://github.com/username/repo/edit/main/docs",
    "showLastUpdated": true,
    "showReadingTime": true,
    "showAuthors": false,
    "showTags": true,
    "versioning": true,
    "i18n": false
  }
}
```

### Feature Options

**editUrl**
- Link to edit page source
- Appears at bottom of each page
- Use GitHub, GitLab, or any Git hosting URL
- Set to `false` to disable

**showLastUpdated**
- Display when page was last modified
- Uses git commit timestamps
- Appears at bottom of page

**showReadingTime**
- Estimate reading time (words per minute)
- Shows "5 min read" at top of page

**showAuthors**
- Display page authors
- Requires author info in frontmatter

**showTags**
- Show tags on pages
- Requires tags in frontmatter

**versioning**
- Enable documentation versioning
- Support multiple doc versions (v1.0.0, v2.0.0)

**i18n**
- Enable internationalization
- Support multiple languages

## Environment Variables

Use environment variables in your config and MDX content.

### In Configuration

```json
{
  "env": {
    "API_BASE_URL": "https://api.example.com",
    "API_VERSION": "v1",
    "CDN_URL": "https://cdn.example.com",
    "SUPPORT_EMAIL": "support@example.com"
  }
}
```

Access in MDX:

```mdx
API endpoint: {process.env.API_BASE_URL}
Current version: {process.env.API_VERSION}
```

### From .env Files

Set values via environment files:

```bash
# .env.local
API_BASE_URL=https://api.example.com
MEILISEARCH_HOST=https://search.example.com
MEILISEARCH_API_KEY=your-key
```

Reference in config:

```json
{
  "env": {
    "API_BASE_URL": "${API_BASE_URL}"
  },
  "search": {
    "meilisearch": {
      "host": "${MEILISEARCH_HOST}",
      "apiKey": "${MEILISEARCH_API_KEY}"
    }
  }
}
```

<Callout type="warning">
Never commit sensitive keys to version control. Use environment variables and add `.env.local` to `.gitignore`.
</Callout>

### Public vs Private Variables

**Private (server-only):**
```
MEILISEARCH_API_KEY=secret-key
DATABASE_URL=postgres://...
```

**Public (exposed to browser):**
```
NEXT_PUBLIC_GA_ID=G-XXXXXXXXXX
NEXT_PUBLIC_API_URL=https://api.example.com
```

Prefix with `NEXT_PUBLIC_` to make variables available in the browser.

## Theme Configuration

Customize visual appearance.

```json
{
  "theme": {
    "defaultMode": "system",
    "respectPrefersColorScheme": true,
    "primaryColor": "#3b82f6",
    "customCss": "/custom.css"
  }
}
```

**defaultMode**
- `"system"` - Match OS preference
- `"light"` - Always light mode
- `"dark"` - Always dark mode

**respectPrefersColorScheme**
- Honor OS color scheme preference
- Set to `false` to ignore OS settings

**primaryColor**
- Main brand color (hex code)
- Used for links, buttons, accents

**customCss**
- Path to custom CSS file
- Add in `public/` folder

See [Customization](/docs/v1.0.0/customization) for detailed theming guide.
